## What Cetele is

A mobile-first **group dhikr tracker** (installable PWA) that uses dopamine hooks + real group accountability to make daily dhikr stick. Full spec: [`docs/PRD.md`](../docs/PRD.md).

Conclusion unchanged: **stay PWA.** Rationale — (1) **Notifications are a solved problem**, the original worry: Web Push works on **Android** (always) and **iOS/iPadOS 16.4+** (since Mar 2023) for **Home-Screen-installed** PWAs; non-installers / older iOS covered by the **email fallback (Resend)**, exactly as D10 specifies (reminders are **server-sent** via Vercel cron / Supabase Edge Fn — the correct pattern anyway). (2) **App store is a 🟡 low–medium blocker, not a dealbreaker**: approval is low-risk (real features clear Apple's 4.2 "minimum functionality" bar; religious/dhikr apps are common), but it adds $99/yr Apple + $25 Google, **1–3 day review cycles** (the real friction — slows iteration during validation), a 30% cut (only if charging), and two more surfaces to maintain. (3) **One codebase, instant updates, no review delays** — best fit for the current **mock-first validation stage (D18)** with no backend yet. **What native would buy** (only if/when needed): frictionless install (no "Add to Home Screen" coaching), store discoverability, slightly more reliable iOS push. **Escape hatch:** if store presence ever becomes necessary, **wrap the same React/web code with Capacitor** (or TWA / PWABuilder) for native APNs/FCM push + store listing — **no Flutter rewrite**. **Known PWA frictions accepted:** iOS install is manual/hidden (needs a one-time coaching screen — the biggest real friction); iOS push is server-only (already the plan); iOS may evict client storage if unused for weeks (irrelevant — data lives in Supabase).
